# Реферальная система Суши-Хаус 39

## Общее описание

Реферальная система состоит из двух частей:
1. **Для всех пользователей** — приглашай друзей, получай бонусы SHC
2. **Для амбассадоров** — двухуровневая комиссия с платежей рефералов (30% / 5%)

---

## Часть 1: Рефералка для обычных пользователей

### Как работает
1. У каждого пользователя есть **реферальная ссылка** (ref_url)
2. Пользователь делится ссылкой через кнопку "Поделиться" в Личном кабинете
3. Друг переходит по ссылке → регистрируется в боте
4. Пользователю начисляется **50 SHC** за каждого нового друга
5. На определённых порогах — дополнительный бонус

### Начисление SHC

**Базовое:** +50 SHC за каждого приглашённого друга

**Бонусы на порогах:**

| Друзей | Бонус | Итого за порог |
|--------|-------|----------------|
| 1 | +50 SHC | 100 SHC |
| 3 | +100 SHC | 250 SHC |
| 5 | +290 SHC | 540 SHC |
| 10 | +490 SHC | 1 280 SHC |
| 50 | +1 500 SHC | 4 780 SHC |
| 100 | +5 000 SHC | 12 280 SHC |
| 500 | +10 000 SHC | 42 280 SHC |
| 1000 | +30 000 SHC | 92 280 SHC |

### Переменные (WATBOT)
- `balance_shc` — текущий баланс SHC
- `referrals` — количество приглашённых друзей
- `ref_ids` — список ID рефералов (защита от повторного учёта)
- `ref_usernames` — список username рефералов
- `ref_usernames_list` — пронумерованный список (для отображения)
- `bonus` — последний начисленный бонус
- `achievement` — текст достижения ("Ты привёл 5 друзей — +290 SHC")

### Реализация (полностью в мини-аппе, SQLite)

- Начисление 50 SHC за нового реферала ✅
- Бонусы на порогах ✅
- Защита от повторного учёта (таблица referral_bonuses) ✅
- Достижения (achievement) ✅
- Кнопка "Поделиться ссылкой" ✅
- Счётчик друзей + баланс SHC ✅
- История начислений SHC с детализацией ✅
- API: `POST /api/register-referral` — регистрация + начисление ✅
- API: `POST /api/get-transactions` — история бонусов ✅

### Что НЕ реализовано
- **Что можно купить за SHC** — не определено (скидка? бесплатный ролл? обмен на рубли?)
- **Магазин за SHC** — интерфейс для траты баланса

## Часть 2: Амбассадорская программа

### Путь амбассадора

1. **Амбассадор** покупает тариф 9990₽ (одноразовый платёж)
2. Получает **реферальную ссылку** (ref_url) для приглашения
3. Делится ссылкой через кнопку "Поделиться ссылкой" в Личном кабинете
4. Приглашённый переходит по ссылке → регистрируется в боте
5. Приглашённый покупает подписку (290 / 490 / 1190₽)
6. **Амбассадору автоматически начисляется 30%** от суммы платежа

### Уровни дохода

| Уровень | Комиссия | Условие |
|---------|----------|---------|
| **Level 1** | 30% от платежей рефералов | Сразу после покупки амбассадорского тарифа |
| **Level 2** | 5% от платежей рефералов своих амбассадоров | Разблокируется после привлечения 10 амбассадоров |

### Примеры начислений (Level 1)

- Реферал оплатил подписку 490₽ → амбассадор получает **147₽**
- Реферал оплатил подписку 1190₽ → амбассадор получает **357₽**
- Реферал оплатил подписку 290₽ → амбассадор получает **87₽**

### Пример Level 2

Амбассадор "А" пригласил 10 амбассадоров. Один из них ("Б") пригласил реферала, который оплатил 490₽:
- Амбассадор "Б" получает 30% = 147₽
- Амбассадор "А" получает 5% = 24.50₽

## Что реализовано (готово)

### База данных (SQLite)
- Таблица `users` — все пользователи, тариф, баланс, связь `invited_by` (кто пригласил)
- Таблица `payments` — все платежи (тариф, сумма, YooKassa ID)
- Таблица `transactions` — все начисления комиссий (сумма, процент, уровень)
- **1475 пользователей** мигрированы из WATBOT

### Бэкенд (API)
| Эндпоинт | Назначение |
|----------|-----------|
| `POST /api/register-referral` | Регистрация связи "кто кого пригласил" |
| `POST /api/get-referrals` | Список рефералов амбассадора (из SQLite) |
| `POST /api/get-transactions` | История начислений + общий заработок |
| `POST /api/migrate-referrals` | Одноразовая миграция из WATBOT |

### Автоматическое начисление комиссий
- При успешном платеже (YooKassa webhook) система:
  1. Записывает платёж в SQLite
  2. Находит, кто пригласил плательщика (`invited_by`)
  3. Если пригласивший — амбассадор → начисляет 30% (Level 1)
  4. Если у амбассадора есть свой пригласивший с ≥10 амбассадорами → начисляет 5% (Level 2)
  5. Обновляет баланс амбассадора в базе

### Личный кабинет (ProfilePage)
Для амбассадоров отображается:
- Золотой бейдж "АМБАССАДОР"
- Счётчики: рефералов / амбассадоров среди них
- Прогресс-бар Level 2 (X / 10 амбассадоров)
- **Блок "Заработок"** — всего / Level 1 / Level 2
- **История начислений** — имя реферала, сумма, процент, дата
- Баланс к выплате
- Кнопка "Поделиться ссылкой"

Для обычных пользователей:
- Блок "Зарабатывайте 30% с каждого платежа"
- Кнопка "Стать амбассадором — 9 990 ₽"

## Что осталось сделать

### 1. Магазин за SHC (нужно решение заказчика)
**Не определено:** что пользователь может сделать с накопленными SHC?
- Скидка на заказ?
- Бесплатное блюдо?
- Обмен на рубли?
- Оплата подписки?

### 2. Привязка реферальной ссылки (критично для амбассадоров)
**Сейчас:** реферальная ссылка (`ref_url`) хранится как переменная в WATBOT. Когда пользователь переходит по ссылке — WATBOT регистрирует реферала.

**Нужно:** при переходе по реферальной ссылке вызывать `POST /api/register-referral` с `telegram_id` приглашённого и `invited_by` (telegram_id амбассадора), чтобы связь записалась в SQLite.

**Варианты реализации:**
- В боте: при старте по реферальной ссылке бот отправляет запрос на `/api/register-referral`
- В мини-аппе: парсить параметр `ref` из URL и отправлять запрос при первом входе

### 3. Механизм вывода средств
**Сейчас:** в интерфейсе написано "Для вывода средств свяжитесь с администратором" (ручной режим).

**В будущем (опционально):**
- Автоматический вывод на карту/кошелёк
- Минимальная сумма для вывода
- Кнопка "Запросить выплату" в личном кабинете

### 4. Уведомления амбассадору
**Не реализовано.** При начислении комиссии амбассадор не получает уведомление. Можно добавить:
- Push в Telegram-бот: "Вам начислено 147₽ за оплату реферала Иван"
- Или просто отображать в личном кабинете (уже сделано)

### 5. Полный отказ от WATBOT
**Текущее состояние:** SQLite — основной источник данных для рефералов и комиссий. WATBOT по-прежнему используется для:
- Хранения контактов и переменных (sync-user читает из WATBOT)
- Реферальных ссылок (ref_url генерируется WATBOT)
- Тегов (Амба, тарифы)

**Для полного отказа нужно:**
- Генерировать реферальные ссылки самостоятельно
- Хранить все данные пользователя только в SQLite
- Оставить WATBOT только для рассылок через бота

## Техническая архитектура

### Стек
- **БД:** SQLite (better-sqlite3), файл `/app/data/sushii.db`
- **Бэкенд:** Express.js на Node 20
- **Фронт:** React (CRA)
- **Деплой:** Docker + Nginx + Let's Encrypt на VPS
- **Оплата:** YooKassa (webhook обрабатывает начисления)

### Ключевые файлы
| Файл | Назначение |
|------|-----------|
| `api/_lib/db.js` | SQLite — таблицы, CRUD, логика комиссий |
| `api/yookassa-webhook.js` | Обработка платежей + начисление комиссий |
| `api/get-referrals.js` | Список рефералов из SQLite |
| `api/get-transactions.js` | История начислений |
| `api/register-referral.js` | Регистрация связи invited_by |
| `api/sync-user.js` | Синхронизация WATBOT → SQLite |
| `src/ProfilePage.js` | Личный кабинет с заработком |

### Схема базы данных

```
users
├── telegram_id (PK)
├── name, phone, tariff
├── invited_by → telegram_id амбассадора
├── balance_shc (баланс к выплате)
├── is_ambassador (0/1)
├── subscription_status, subscription_start, subscription_end
└── created_at, updated_at

payments
├── id (PK)
├── telegram_id → users
├── tariff, amount, months
├── yookassa_payment_id
└── created_at

transactions (комиссии)
├── id (PK)
├── ambassador_id → users
├── referral_id → users
├── payment_id → payments
├── payment_amount, commission_amount, commission_percent
├── level (1 или 2)
└── created_at
```
