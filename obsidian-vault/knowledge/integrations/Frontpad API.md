# Frontpad API

Frontpad — CRM-система ресторана. Используется для:
1. Создания заказов
2. Хранения каталога товаров (артикулы)

Клиент: `api/_lib/frontpad.js`
Переменная: `FRONTPAD_SECRET`

## Создание заказа

Эндпоинт: `POST /api/create-order`
Файл: `api/create-order.js`

Телефон берётся из SQLite (не из формы!) — нормализуется по правилу [[нормализация номера телефона]].

Для создания заказа нужен **affiliate ID** — ID точки самовывоза. Определяется через [[Nearest Store — ближайшая точка]]:
```
POST /api/nearest-store  →  { affiliate }  →  Frontpad order
```

Магазины в `config/stores.json`:
| Название | Frontpad affiliate |
|---------|-------------------|
| Гагарина | ... |
| Согласия | ... |
| Автомобильная | ... |
| Гурьевск | ... |

## Артикулы товаров (SKU)

Каждый товар в Frontpad имеет уникальный числовой артикул. Он же хранится в JSON-каталогах как поле `sku`.

**Тарифы подписки:**
| Тариф | Frontpad артикул |
|-------|----------------|
| 290 | 1177 |
| 490 | 1050 |
| 1190 | 1178 |
| 9990 (Амбассадор) | 1215 |

## Каталог товаров

Выгрузка всего каталога: `npm run fetch:products`
Файл выгрузки: `Товары.csv` (кодировка cp1251, разделитель `;`)

Структура CSV: `Категория;Название;Цена;Артикул`

**342 позиции** в 13 категориях (по состоянию на 2026-04-07).

→ Единый справочник с сопоставлением: `data/catalog-master.csv`
→ Скрипт генерации: `scripts/build-catalog-master.py`

## Синхронизация каталогов

При включении/выключении товара в админке синхронизируются связанные каталоги:

```js
['rolls', 'rolls-sub', 'rolls-490']    // холодные роллы
['zaproll', 'zaproll-sub']             // запечённые
['sets', 'sets-sub', 'sets-490']       // сеты
```

→ `api/admin-products.js`

## Webhook бота

`POST /api/order` — это URL для уведомлений от watbot.ru (устаревший бот).
**Не трогать** — это не API, это webhook endpoint для внешнего сервиса.
