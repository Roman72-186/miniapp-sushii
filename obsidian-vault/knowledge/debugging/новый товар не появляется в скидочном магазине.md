# Ловушка: новый товар в zaproll.json не появляется в скидочном магазине

## Симптом
Добавил товар в `public/запеченные роллы/zaproll.json` — в обычном магазине появился, в скидочном (`/discount-shop`) нет.

## Причина
Скидочный магазин загружает данные из **отдельных** subscription-каталогов:
- Запечённые → `public/подписка запеченные/zaproll-sub.json`
- Холодные → `public/подписка роллы/rolls-sub.json`
- Сеты → `public/подписка сеты/sets-sub.json`

Admin-синхронизация (при включении/выключении товара) работает только для **уже существующих** позиций в обоих каталогах. Новый товар нужно добавлять вручную в оба файла.

## Что нужно для добавления в zaproll-sub.json
Нужен **Frontpad subscription SKU** — отдельный ID товара в подписочном меню Frontpad. Он **отличается** от обычного SKU.

### Как найти subscription SKU
1. Открыть `Товары.csv` (кодировка **cp1251**) в разделе «Запеченные зап. (подписка)»
2. Или спросить у пользователя (он знает из Frontpad-панели)

### Структура записи в zaproll-sub.json
```json
{"name": "Запечённый лосось файр", "price": 610, "sku": "11155"}
```
Цена — **базовая** (до скидки). Скидка 30% считается на фронтенде.

## Полный чеклист добавления нового запечённого ролла
1. ✅ Добавить в `public/запеченные роллы/zaproll.json` (обычный SKU, обычная цена)
2. ✅ Добавить в `public/подписка запеченные/zaproll-sub.json` (subscription SKU, та же цена)
3. ✅ Проверить override на VPS: `docker exec miniapp-sushii-app-1 ls /app/data/products/`
4. ✅ Если override есть — добавить товар и туда через `docker exec`
5. ✅ Добавить фото в `public/new_roll/` и сделать `git add` (untracked файлы не деплоятся!)
6. ✅ Добавить маппинг в `src/config/imageMap.js`
