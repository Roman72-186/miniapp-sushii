# Баг нормализации э→е в imageMap

**Статус:** ✅ Исправлен 2026-04-07

## Симптом

14 товаров с буквой `э` в названии показывали логотип вместо фотографии:
- Эби гриль, Фила эби гриль, Спайси эби маки
- Краб **дуэт** гриль, Калифорния **дуэт**
- Запечённый **Мидуэй**
- Суши-**сэ**ндвич с лосасем
- и др.

## Причина

Функция `normalizeName()` заменяет `э→е`:
```js
.replace(/э/g, 'е')
```

При поиске `"Спайси эби маки"`:
1. normalize → `"спайси еби маки"` (э стала е)
2. Ищем в IMAGE_MAP ключ `"спайси еби маки"`
3. Ключ хранится как `"спайси эби маки"` (с э!)
4. Точного совпадения нет
5. Частичное совпадение тоже не работает (`"еби" ≠ "эби"`)
6. Возвращается `"/logo.jpg"`

## Диагностика

Проверили нормализацию напрямую:
```python
normalize("Краб дуэт гриль") → "краб дует гриль"   # э→е
# но в imageMap ключ: 'краб дуэт гриль'
```

## Исправление

В `src/config/imageMap.js` добавили предвычисленный словарь с нормализованными ключами:

```js
const _NORMALIZED_MAP = Object.fromEntries(
  Object.entries(IMAGE_MAP).map(([k, v]) => [normalizeName(k), v])
);
```

И изменили `getProductImage` чтобы использовал `_NORMALIZED_MAP` вместо `IMAGE_MAP`.

Теперь ключи нормализуются ОДИН РАЗ при загрузке модуля.

## Урок

При добавлении ключей в IMAGE_MAP — можно писать с `э` или `е`, оба варианта работают благодаря предвычислению. Но нужно помнить: ключи **нормализуются**, поэтому `"эби"` и `"еби"` в IMAGE_MAP — это одна и та же запись.
