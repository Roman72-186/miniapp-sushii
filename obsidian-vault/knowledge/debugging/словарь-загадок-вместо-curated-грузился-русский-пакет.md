# Ловушка: словарь загадок игры грузился из русскоязычного пакета, а не из curated файла

## Симптом
Игроки жаловались: в игре `/game` попадались архаичные, книжные, редкие слова (например, «врата», «жатва», «перст», «холоп», «шулер», «яство»). Curated `api/game-words.json` правился и чистился — жалобы продолжались.

## Причина
В `api/_lib/db.js:132` и `api/_lib/db-pg.js:66` (обе копии — SQLite и PostgreSQL версии) seed таблицы `game_word_dictionary` выполнялся через **`loadFiveLetterWords()` из `api/_lib/game-dictionary.js`**, который возвращает **6074 слова** из npm-пакетов `russian-words` + `wordlist-russian` — включая архаизмы, диалектизмы, редкие термины.

Комментарий в коде был ложью: `// Sync словаря: только curated-список` — по факту грузилось 6074 слов, а не curated 480.

Curated `api/game-words.json` **не использовался** вообще.

## Как это попало в прод
Коммит 2026-04-17 (сессия 5) «словарь russian-words + авто-синк» — тогда был добавлен `loadFiveLetterWords()` для расширения `VALID_WORDS` (валидации вводимых гессов). При этом seed БД тоже перевели на эту функцию, хотя должны были оставить на curated. Комментарий написали «только curated», но реализовали иначе. Ошибка застряла, потому что никто не проверил в продакшн-БД содержимое `game_word_dictionary`.

## Правильная архитектура (как сейчас работает после фикса 2026-04-21)

| Цель | Источник | Количество |
|------|----------|------------|
| **Загадки** (что игрок отгадывает) | `api/game-words.json` → `game_word_dictionary` | curated 480 |
| **Валидация гессов** (введённые слова принимаются/отклоняются) | `loadFiveLetterWords()` → `VALID_WORDS` Set в `api/game-guess.js` | 6074 |

Два разных словаря — это намеренно. Игрок может вводить любое известное слово (включая слегка редкие — 6074 годятся), но **загадывается** только из тщательно отобранного списка.

## Как проверить в проде
```bash
ssh root@64.188.63.249
docker exec miniapp-sushii-app-1 sqlite3 /app/data/sushii.db "SELECT COUNT(*) FROM game_word_dictionary;"
# Должно быть 480 (curated), а не 6074
```

Также: при старте контейнера должен быть лог `[db] syncGameDictionary (curated): 480 слов` (с суффиксом `(curated)`) или `[db-pg] syncGameDictionary (curated): 480 слов`.

## Правила на будущее
1. **Не доверять комментариям** «только curated» — проверять по коду.
2. При любой работе с `game_word_dictionary` — проверить, что грузится из `api/game-words.json`, а не из `loadFiveLetterWords()`.
3. Если `loadFiveLetterWords()` всё же используется — это **VALID_WORDS для валидации гессов**, а не для загадок.
4. После любой правки `game-words.json` словарь автоматически пересинхронизируется при рестарте контейнера (старые удаляются, новые добавляются, активные игры с потерянным словом сбрасываются).

## Сброс активных игр
При смене curated-словаря у пользователей в колонке `users.game_current_word` может остаться слово, которого больше нет в `game_word_dictionary`. После sync-блока теперь работает UPDATE, сбрасывающий у таких юзеров `game_current_word`, `game_word_status`, `game_session_id` — на `/api/game-stats` им назначится новое слово из актуального curated.

## Связанные файлы
- `api/_lib/db.js:130-160` — sync SQLite + сброс активных игр
- `api/_lib/db-pg.js:64-90` — sync PostgreSQL + сброс активных игр
- `api/game-words.json` — curated список загадок (480 слов)
- `api/_lib/game-dictionary.js` — `loadFiveLetterWords()` для валидации гессов (6074 слов)
- `api/game-guess.js` — принимает гесс, проверяет через `VALID_WORDS`
