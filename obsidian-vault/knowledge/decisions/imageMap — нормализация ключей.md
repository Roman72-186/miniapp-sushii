# imageMap — нормализация ключей

## Файл

`src/config/imageMap.js`

## Как работает поиск картинки

```js
getProductImage("Краб дуэт гриль")
    ↓
normalizeName() → "краб дует гриль"  // э→е, lowercase
    ↓
_NORMALIZED_MAP["краб дует гриль"]   // предвычисленный словарь
    ↓
"/new_roll/Краб дуэт гриль.jpg"
```

## Ключевое решение: предвычисленный `_NORMALIZED_MAP`

До 2026-04-07 `IMAGE_MAP` хранил ключи «как написано» (с `э`), а `normalizeName()` превращала `э→е`.
Это означало: поиск `"краб дует гриль"` никогда не находил ключ `"краб дуэт гриль"`.

**Исправление:**
```js
const _NORMALIZED_MAP = Object.fromEntries(
  Object.entries(IMAGE_MAP).map(([k, v]) => [normalizeName(k), v])
);
```

Теперь ключи тоже нормализуются один раз при старте.

→ [[баг нормализации э→е в imageMap]]

## Правила написания ключей в IMAGE_MAP

Писать можно с `э` или с `е` — не важно, всё нормализуется.
Но для читаемости — пишем как в реальном названии товара.

## Частичное совпадение

Если точного совпадения нет — ищем подстроку:
```js
if (normalized.includes(key) || key.includes(normalized))
```

Это позволяет найти картинку если название чуть отличается.
⚠️ Может дать ложное совпадение для коротких ключей (напр. `'краб'`).

## Структура IMAGE_MAP (секции)

1. PNG-файлы с английскими именами (старые)
2. JPG-файлы с русскими именами
3. Дополнительные алиасы (префиксы «ролл»)
4. Сеты
5. Добавки и соусы
6. Гунканы
7. Новые фото из `new_roll/` (добавлено 2026-04-07)
