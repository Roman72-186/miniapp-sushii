# 2026-04-13 (3) — Выгрузки Watbot × DB для рассылки возврата

## Цель сессии

Собрать пул пользователей для **рассылки напоминания оплатить подписку**:
- Взять из Watbot контакты, у которых есть сохранённая ссылка на оплату (переменная `link`)
- Вычесть тех, кто уже оплатил подписку (активные в нашей БД)
- Вычесть тех, у кого нет `telegram_id` (некому отправлять)
- Получить пригодный к рассылке файл

## Что сделано

### 1. Разобрался с Watbot API

Пользователь дал ссылку на документацию: <https://docs.watbot.ru/rabota-s-api/kontakty>

Ключевой эндпоинт: **`GET /api/v1/getContacts`**
- Параметры query: `api_token`, `bot_id`, `count` (макс 500), `page`
- Пагинация: `response.meta.last_page` + `response.data`
- Переменные контакта лежат в `contact.variables` — массив объектов `{ name, value }`

В архиве `_archive/scripts/sync-all-from-watbot.js` нашёл **`BOT_ID = 72975`** — это наш активный бот. Также узнал формат `variables` из функции `getVar(variables, ...names)` — переменные берутся по имени.

### 2. `scripts/fetch-watbot-users-with-link.js` — выгрузка всех с link

Читает все контакты через пагинацию, фильтрует по наличию непустой переменной `link` (имя конфигурируемое через argv). Rate limit 300 мс между страницами.

**Результат на проде:**
- Всего контактов в Watbot: **1522**
- С заполненной `link`: **536**
- Файл: `/app/data/watbot-users-with-link.json` (216 KB)

**Важная находка:** значение `link` — это **ссылки YooMoney/YooKassa на checkout**, формата `https://yoomoney.ru/checkout/payments/v2/contract?orderId=...`. Watbot сохранял ссылку на последнюю созданную сессию оплаты для каждого контакта.

### 3. `scripts/merge-watbot-link-with-db.js` — сопоставление с БД

Для каждого из 536 watbot-контактов проверяет: есть ли в `users` по `telegram_id`, активна ли подписка (по `subscription_status` + парсинг дат `DD.MM.YYYY`).

Разбивка на 3 корзины:
- `matched_not_active` — в БД, подписка не активна (целевая группа)
- `matched_active` — в БД, подписка активна (исключены)
- `not_in_db` — в Watbot есть, в нашей БД по `telegram_id` не найдены

**Результат:**
| Корзина | Кол-во |
|---|---:|
| В БД, НЕ активные | **98** |
| В БД, активные (исключены) | 56 |
| Не в БД | **382** |

Странно большое «не в БД» (71%): много старых watbot-контактов, которые либо никогда не дошли до оплаты, либо перешли в web-регистрацию под `web_*` ID (в этом случае по `telegram_id` не найдутся — теоретически можно искать по телефону из `variables`, но это уже отдельная задача).

Файл: `/app/data/watbot-link-merged.json`.

### 4. `scripts/fetch-db-users-no-active-sub.js` — выгрузка неактивных из БД

По запросу «собрать всех с телеграм id без активной подписки». Отдельно от Watbot-контекста — только наша БД.

Фильтры:
- `telegram_id` — числовая строка (не `web_*`)
- Подписка не активна (по тому же `isSubscriptionActive`)

**Результат:**
| | Кол-во |
|---|---:|
| Всего в `users` | 261 |
| С числовым `telegram_id` | 162 |
| — из них БЕЗ активной подписки | **102** |
| — из них с активной подпиской | 60 |
| `web_*` (исключены) | 99 |

Файл: `/app/data/db-users-no-active-sub.json` (60 KB).
**Скачан локально:** `data/db-users-no-active-sub.json`.

### 5. Общая статистика по `users`

| Метрика | Значение |
|---|---:|
| Всего пользователей | 261 |
| Амбассадоры | 1 |
| Активные подписки | 84 |
| Неактивные | 177 |

**Активные по тарифам:** 290 → 31, 490 → 28, 1190 → 14, без тарифа → 11.
**Неактивные по тарифам:** без тарифа → 124 (лиды, никогда не платили), 490 → 23, 290 → 17, 1190 → 13.

Ключевой инсайт: **124 лида зарегистрировались, но не оплатили ни разу** — это основной потенциал для активации, отдельная задача.

### 6. Объединённая выгрузка: Watbot × активные → `watbot-link-tg-only.json`

По финальному запросу: взять 536 watbot-контактов с `link`, вычесть 56 активных в нашей БД, оставить только с `telegram_id`.

Так как merge уже был готов в `watbot-link-merged.json`, просто склеил две корзины (`matched_not_active` + `not_in_db`) в одну. Проверка `telegram_id` — все 480 уже с числовым tg_id (Watbot-контакты по определению из Telegram).

**Итоговый файл:** `data/watbot-link-tg-only.json` (237 KB, **480 записей**).

Структура каждой записи:
```json
{
  "id": 6160280,
  "telegram_id": "859665890",
  "telegram_username": "mini_girl0_0",
  "name": "🧸",
  "link": "https://yoomoney.ru/checkout/payments/v2/contract?orderId=...",
  "db": { "tariff": null, "subscription_status": "неактивно", "subscription_end": "04.01.2026", ... }
}
```

Поле `db` присутствует только у 98 записей (те что нашлись в БД). У остальных 382 — только watbot-поля + `reason: 'not_found'`.

## Открытые вопросы / что осталось

- **Рассылка по 480 через Telegram Bot API** — файл `data/watbot-link-tg-only.json` готов: у каждого есть `telegram_id` и сохранённая ссылка на YooMoney checkout. Нужно согласовать **текст сообщения** с заказчиком и написать скрипт отправки. Risk: часть из 382 «не в БД» — это боты/заблокированные/неактивные в Telegram аккаунты. Bot API вернёт 403 для них — нужна обработка ошибок.
- **124 лида без тарифа** — зарегистрировались, но ни разу не платили. Отдельная задача активации — возможно welcome-flow или промокод.
- **Поиск по телефону вместо tg_id** — для 382 «не в БД» можно попробовать найти по телефону из `variables.phone` (Watbot) → `users.phone`. Это вытащит web_-юзеров, которые регистрировались заново.
- **Битый `data/db-users-all.csv`** — если понадобится полная CSV-выгрузка, перекачать без `2>&1` слияния потоков (я начал, но прервали).
