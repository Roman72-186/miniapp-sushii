# Сессия 2026-04-08 (4) — Запечённый лосось файр в скидочном магазине

## Задача
Товар «Запечённый лосось файр» добавлен в `zaproll.json` ранее, но не отображается в скидочном магазине `/discount-shop`.

## Диагностика

### Причина 1: отсутствие в zaproll-sub.json
`DiscountShopPage.js` загружает товары из `/подписка запеченные/zaproll-sub.json`, а не из `zaproll.json`.
«Запечённый лосось файр» в `zaproll-sub.json` **никогда не был добавлен** — ни при создании каталога, ни при добавлении товара.

Для добавления нужен **Frontpad subscription SKU** (не равен обычному SKU 317).
- Нашли через `Товары.csv` (кодировка cp1251) — в разделе «Запеченные зап. (подписка)» товара нет.
- В разделе «Роллы по подписке **» (бесплатные подарки) он есть — SKU **1005**.
- Пользователь предоставил subscription SKU для скидочного каталога: **11155** (5 цифр).

### Причина 2: override-файл на VPS
После добавления в `zaproll-sub.json` и деплоя товар всё равно не появился.

Причина: на VPS в Docker-volume существует файл-override:
```
/app/data/products/подписка запеченные/zaproll-sub.json
```
Этот файл был создан администратором через панель и **перекрывает `build/`** (Express отдаёт `data/products/` первым).

В override-файле все `enabled: undefined` (никаких реальных изменений), но новый товар в нём отсутствовал.

**Фикс**: добавили товар напрямую в override через `docker exec`.

### Причина 3: фото не в git
После появления товара — нет фотографии. Файл `/public/new_roll/Запеченный лосось файер.jpg` был **untracked** (не добавлен в git). При деплое не попал в `build/`.

**Фикс**: `git add` + commit + push + rebuild.

## Что сделано
1. Добавлен в `public/подписка запеченные/zaproll-sub.json` (sku: 11155, price: 610)
2. Добавлен в `/app/data/products/подписка запеченные/zaproll-sub.json` на VPS напрямую
3. Закоммичено фото `public/new_roll/Запеченный лосось файер.jpg`
4. Амбассадорские комиссии (`processCommissions`) — зафиксированы как намеренно отключённые в `yookassa-webhook.js`
