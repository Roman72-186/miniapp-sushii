# 2026-04-18 (4) — Fix: web ID при ручном добавлении пользователя

## Что сделано

Исправлена генерация `telegram_id` при ручном добавлении пользователя через админку.

**Проблема:** `api/admin/add-user-manual.js` присваивал статичный ID `web_7XXXXXXXXXX` (web_ + телефон). При последующем входе пользователя через веб `set-password.js` находил его по телефону и оставлял с этим нестандартным ID.

**Исправление:** строка 61 в `api/admin/add-user-manual.js`:

```js
// БЫЛО:
const telegram_id = existingUser ? existingUser.telegram_id : `web_${normalizedPhone}`;

// СТАЛО:
const telegram_id = existingUser
  ? existingUser.telegram_id
  : 'web_' + Date.now() + '_' + Math.random().toString(36).slice(2, 9);
```

## Файлы изменены

- `api/admin/add-user-manual.js` — строка 61

## Решения

- Существующие пользователи не затрагиваются (ветка `existingUser.telegram_id`)
- Новые пользователи получают стандартный формат `web_{ts}_{random}`, совместимый с веб-регистрацией
- Деплой не потребовался (изменение в одной строке, без миграций)
