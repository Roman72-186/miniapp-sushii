# 2026-04-20 (2) — подписи подарков + раздельная статистика подписки/заказы

## Что сделано

### 1. Подписи подарков в корзине
- `CartPanel.js` — меняем текст:
  - `giftSource === 'promo'` → **«По промокоду»** (было «Подарок по промокоду»)
  - `giftSource === 'threshold2500'` → **«За горячий чек»** (было «Подарок к заказу»)
  - `subscription` — оставили как «Подарок по подписке».

### 2. Логирование промо/порогового подарка в заказ
- `CheckoutForm.js` — при формировании payload для `/api/create-order` каждому продукту добавлено поле `gift_source` (из `item.giftSource`). `promo_code` уже передавался.
- `api/create-order.js` — валидирует `promo_code` (regex `^[A-Za-z0-9]{1,20}$`), вычисляет `hasPromoGift` / `hasThresholdGift` из `products[].gift_source`, пробрасывает в расширенный `insertOrder`. `products_json` теперь хранит `giftSource` у каждой позиции.

### 3. Миграция БД (идемпотентная, runtime)
Новые колонки в `orders` — добавляются через `ALTER TABLE ... ADD COLUMN` при старте:
- `promo_code TEXT` (NULL)
- `has_promo_gift INTEGER DEFAULT 0` (BOOLEAN в PG)
- `has_threshold_gift INTEGER DEFAULT 0`

Файлы: `api/_lib/db.js`, `api/_lib/db-pg.js`.

### 4. Новые хелперы в db-модулях
- `getOrdersStats(sinceIso)` → `{ orders, revenue, promoGifts, thresholdGifts }`
- `getPaymentsStats(sinceIso)` → `{ count, revenue }` — **добавлено вторым коммитом**, чтобы считать выручку по подпискам отдельно.
- `getPaymentsCount(sinceIso)` — оставлен (не используется после рефакторинга, но экспортирован).
- `getOrdersDaily(days)` → массив `[{ date, count, revenue }, ...]` за N дней.

### 5. Расширение admin-stats
`api/admin-stats.js` — старый ответ сохранён целиком, добавлены:
- `periods: { '1d', '7d', '15d', '30d' }` каждый с:
  - `orders` / `ordersRevenue` — заказы магазина
  - `newSubs` / `subsRevenue` — подписки (payments)
  - `promoGifts` / `thresholdGifts`
- `ordersDaily: [{ date, count, revenue }, ...]` за 30 дней для графика.

### 6. UI статистики (`AdminPage.js`, вкладка «◉ Статистика»)
- Блок «За период» — 4 карточки (1/7/15/30 дней). В каждой:
  - Верхняя строка: **Подписки** (₽ + кол-во оплат) vs **Заказы** (₽ + кол-во заказов) — грид 1×2.
  - Нижняя строка: Промокод 102030 vs За чек ≥2500 ₽ — грид 1×2.
- Inline SVG bar-chart заказов по дням (30 дней), подсказка через `<title>` с датой/суммой.

## Важный нюанс

Исторические заказы до деплоя имеют `has_promo_gift=0 / has_threshold_gift=0 / promo_code=NULL` — статистика по подаркам стартует с момента выкатки, ретро-данных нет. Подписки и выручка из `payments` — считаются корректно за всё время.
