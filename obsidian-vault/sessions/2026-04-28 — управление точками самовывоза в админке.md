# 2026-04-28 — Управление точками самовывоза в админке

## Что сделано

Добавлена возможность включать/выключать точки самовывоза из панели администратора. Изменения вступают в силу сразу — выключенная точка пропадает из формы оформления заказа.

## Архитектура

**Хранение состояния:** `data/store-states.json` в Docker volume (по аналогии с `data/products/upsell.json`).
```json
{ "points": { "1": true, "2": true, "3": true, "4": false } }
```
Файл создаётся при первом toggle. Если отсутствует — все точки считаются включёнными (graceful fallback).

## Файлы созданы

- `api/stores-config.js` — публичный GET `/api/stores-config` → `{ points: {"1": true, ...} }` (без auth)
- `api/admin-stores.js` — admin GET/PUT `/api/admin/stores` (с checkAuth)

## Файлы изменены

- `server.js` — добавлены два роута
- `src/components/CheckoutForm.js`:
  - `useState(null)` для `enabledPoints`
  - `useEffect` на mount → `fetch('/api/stores-config')`
  - `activePoints = PICKUP_POINTS.filter(p => enabledPoints[p.id] !== false)` 
  - `useEffect` для сброса `pickupPoint` на первую активную точку
  - В JSX: `PICKUP_POINTS` → `activePoints`, fallback «Самовывоз временно недоступен» при пустом списке
- `src/AdminPage.js`:
  - State: `storesState`, `storesLoading`, `storesSaving`
  - `loadStores()` и `toggleStore()` функции
  - Загрузка при `tab === 'stores'` в существующем useEffect
  - Новая вкладка «📍 Точки самовывоза» в бургер-меню
  - JSX рендер вкладки: список 4 точек с кнопками «Включить» / «Выключить»
