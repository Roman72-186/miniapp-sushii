# Сессия 2026-04-08 (3) — фикс видимости «История подарков» в профиле

## Задача
Пользователь сообщил: «Историю заказов я так и не нашел у пользователя. Нужно посмотреть и обязательно в порядке добавить все это.»

## Проблема

В `src/ProfilePage.js` секция «История подарков» инициализировалась с `giftHistory = null` и показывалась только при `giftHistory !== null`. Это приводило к тому, что секция была **невидима** в следующих случаях:

1. Запрос `/api/get-gift-history` ещё не завершился
2. Запрос упал с сетевой ошибкой (catch → пустой блок `catch(() => {})`)
3. Ответ пришёл, но `data.success === false` или `data === null`

В любом из этих случаев `giftHistory` оставался `null` и секция никогда не рендерилась.

## Исправление

**`src/ProfilePage.js`:**

1. В `useEffect` добавлен fallback — устанавливает `setGiftHistory([])` при ошибке или отсутствии `telegramId`
2. Секция убрана из условия `{giftHistory !== null && ...}` — теперь **всегда рендерится**
3. Добавлено состояние «Загрузка...» пока `giftHistory === null`

```js
// До:
useEffect(() => {
  if (!telegramId) return;
  fetch(...)
    .then(r => r.ok ? r.json() : null)
    .then(data => { if (data?.success) setGiftHistory(data.history); })
    .catch(() => {});
}, [telegramId]);

// После:
useEffect(() => {
  if (!telegramId) {
    setGiftHistory([]);
    return;
  }
  fetch(...)
    .then(r => r.ok ? r.json() : null)
    .then(data => { setGiftHistory(data?.success ? data.history : []); })
    .catch(() => setGiftHistory([]));
}, [telegramId]);
```

```jsx
// До: {giftHistory !== null && (...)}
// После: всегда показываем секцию, с loading/empty/list стейтами
<div className="shop-profile__section">
  <div className="shop-profile__label">🎁 История подарков</div>
  {giftHistory === null ? (
    <div>Загрузка...</div>
  ) : giftHistory.length === 0 ? (
    <div>Подарков пока не было</div>
  ) : (
    <ul>...</ul>
  )}
</div>
```
