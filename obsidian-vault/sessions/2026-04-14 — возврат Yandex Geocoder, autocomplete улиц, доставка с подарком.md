# 2026-04-14 — Возврат Yandex Geocoder, autocomplete улиц, доставка при подарке+дозаказе

## Цель сессии

Вернуть фичу «подарок + дозаказ → доступна доставка», которую я откатывал в прошлой сессии из-за ломаного routing. Исправить root cause проблем с routing-ом — перейти обратно на Yandex Geocoder + добавить autocomplete адресов, сделать номер дома обязательным.

### 2. Возврат `api/_lib/geocoder.js` на Yandex

Полностью переписал модуль:
- Читает `YANDEX_GEOCODER_API_KEY`, `CITY_NAME`, `CITY_BBOX` из env
- `CITY_BBOX` в едином формате `lon1,lat1,lon2,lat2` конвертируется в Yandex-формат `lon1,lat1~lon2,lat2`
- Функция `geocode(address)` — одиночный запрос, возвращает `{ lat, lon, formatted, precision, kind }`
- Новая функция `suggest(query, limit)` — для autocomplete
- Обе функции через общий `yandexRequest()` c bbox + rspn=1

### 3. Новая ручка `POST /api/address-suggest`

Простой обработчик в `api/address-suggest.js`:
- Принимает `{ query }`
- Вызывает `suggest(query, 7)`
- Возвращает `{ success: true, suggestions: [...] }`
- Зарегистрировано в `server.js`

### 5. Итерации по `suggest()` — эвристики для Yandex

Yandex Geocoder API — не классический suggest-сервис. На короткие голые токены возвращает мусор: сам город, аэропорт Храброво, ж/д вокзалы, областные трассы `27А-002/27К-256`. Пришлось несколько итераций до правильного поведения:

**Итерация 1** (коммит `6f7d71b`): добавил `ул.` префикс для всех запросов. Проблема: многословные запросы типа «юрия гагарина» с префиксом сбивались — Yandex искал «улицу Юрия» вместо «улицы Юрия Гагарина».

**Итерация 2** (коммит `059eedd`): сделал 2 параллельных запроса — с префиксом и без, объединял результаты. Проблема: чистые запросы без префикса возвращали те самые трассы `27А-002`, которые попадали в итог.

**Итерация 3** (коммит `6540870`): финальная стратегия:
- Если в query есть тип (`улица`, `проспект`, `переулок`, `шоссе`, `бульвар`, `набережная`, и т.д.) — шлём как есть
- Если query одно слово — добавляем `ул.`
- Если query много слов без типа — шлём как есть
- Только один запрос к Yandex вместо двух

**Итерация 4** (коммит `18a727f`): добавил фильтр «только `, Калининград,` в formatted» — отсекает областные трассы и соседние посёлки Гурьевского округа.

Итоговая логика в `suggest()`:
```
query.length >= 2
→ определить финальный query по трём правилам
→ rawSuggest(finalQuery, limit*3)
→ фильтр: kind ∈ {street, house} И formatted.includes(", Калининград,")
→ дедуп по formatted
→ slice(0, limit)
```

### 7. Проблема с Windows bash curl и кодировкой

При финальных проверках через `curl` с локальной машины все запросы с кириллицей возвращали мусор (трассы, пустой список) — даже когда серверная логика правильная. Оказалось — Windows Git Bash портит кодировку кириллицы в `-d '{"query":"багратиона"}'`, тело уходит в cp1251/mojibake. Проверил: через `docker exec ... node -e` с `JSON.stringify()` всё чисто (byte-level UTF-8). В реальном браузере JSON body всегда UTF-8, так что для пользователей всё работает. Это нюанс тестирования.

## Решения

- **Не переходили на Dadata/Yandex Suggest API** — оказалось, что Yandex Geocoder API сам по себе справляется с подсказками через `results=N` параметр, если правильно подготовить query и отфильтровать результаты. Один API, один ключ, не нужна отдельная активация. Пришлось поигрывать с эвристиками, но итог — минимум зависимостей.
- **Routing Haversine оставлен как есть** — фикс геокодера делает его точнее, но бизнес-логика районов не реализована. Юзер явно сказал не трогать.
- **Не делал fallback-search «большая окружная»** для «4-я окружная» — это костыль, лучше научить юзера или добавить в UI подсказку «не нашли? попробуйте с «Большая»». Оставлено на усмотрение.
- **`streetConfirmed` флаг делает форму строгой** — юзер обязан выбрать пункт из списка. Это гарантирует валидный геокод и точный nearest-store. Ценой — небольшое неудобство UX если Yandex не нашёл.

## Открытые вопросы / что осталось

- **Ручная проверка в браузере** — обязательно: собрать корзину «обычный ролл + подарочный ролл», начать печатать улицу, выбрать из списка, ввести дом, убедиться что nearest-store показывается, отправить заказ и проверить что Frontpad получил `sale: 'delivery'` + правильный `affiliate`.
- **Routing Багратиона → Гагарина** — отдельной задачей, когда будут бизнес-правила от Юлии по районам.
- **«4-я Окружная» UX** — юзеры обычно пишут именно так, а Yandex знает только «4-я Большая Окружная». Возможные решения: (а) добавить алиасы в `suggest()`, (б) добавить подсказку в форме «если не нашли — попробуйте «Большая»», (в) fallback-поиск с дописанным словом.
- **Битый `data/db-users-all.csv`** из прошлой сессии — не удалил, так и лежит в локальной `data/`. Когда понадобится полная CSV — перекачать.
