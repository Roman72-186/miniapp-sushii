### 3. Фикс: реферальная ссылка для новых (незалогиненных) пользователей
**Коммит:** последний в сессии

**Баг:** новый пользователь кликал `/?invited_by=xxx`, проходил логин → `window.location.href = '/'` → `?invited_by=` терялся, реферальная связь никогда не записывалась.

**Фикс:** при обнаружении `?invited_by=` в URL — сразу сохранять в `sessionStorage` под ключом `pending_invited_by`. После логина (когда `telegramId` появляется) — читать из `sessionStorage`, вызывать `/api/register-referral`, удалять.

**Изменён файл:** `src/UserContext.js`
- Добавлена константа `PENDING_REFERRAL_KEY = 'pending_invited_by'`
- `?invited_by=` убирается из URL сразу при заходе (чтобы не торчал)
- После логина `telegramId` появляется → `useEffect` срабатывает → читает `sessionStorage` → регистрирует → удаляет

- `register-referral.js` — не трогали
- `yookassa-webhook.js` — не трогали

## Архитектура реферальной системы (итоговая)

1. Новый пользователь кликает `/?invited_by=XXX`
2. `UserContext` → сохраняет `XXX` в `sessionStorage`
3. Пользователь регистрируется/логинится → `telegramId` появляется
4. `UserContext` → `POST /api/register-referral` → `setInvitedBy` в БД
5. Пользователь оплачивает подписку → `yookassa-webhook` → `processReferralSHC` → пригласившему +20% от суммы в SHC

## Открытые вопросы
- `apply-partner-code.js` не вызывает `processReferralBonus` — это ок, но и `processReferralSHC` не вызывается для ретроактивных платежей через партнёрский код (там своя логика с `getLastPayment` + `updateBalance` напрямую)
