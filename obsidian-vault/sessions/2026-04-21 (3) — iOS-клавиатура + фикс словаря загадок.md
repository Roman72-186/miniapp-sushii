# 2026-04-21 (3) — iOS-клавиатура игры + критфикс источника словаря загадок

## Что сделано

### 1. UX long-press: drag-to-select → tap-выбор
Была проблема: при удержании «Е» появлялся popup, но палец, двигаясь к «Ё», провоцировал **скролл страницы** (тач-жест drag). Переработал UX:
- Корневой элемент клавиши с `altChars` теперь `<div role="button">` (раньше `<button>`) — чтобы внутри popup'а были валидные вложенные `<button>`-пункты.
- Сценарий: зажал → popup появился и **остаётся открытым** → палец можно отпускать → отдельный **тап** по «Е» или «Ё» → выбор и закрытие popup. Тап вне popup → закрытие без ввода.
- `e.preventDefault()` на pointerdown + `touch-action: manipulation` + `-webkit-tap-highlight-color: transparent` — убрали скролл и синий блик.

### 2. iOS-стиль раскладки клавиатуры
Полностью переделал визуал под iOS dark keyboard:
- Подложка клавиатуры `#1c1c1e`, padding `8px 3px 10px`, radius 6px.
- Клавиши: светло-серые `#6b6b6d`, при нажатии `#a8a8a8`, тень `0 1px 0 rgba(0,0,0,0.75)` снизу.
- Шрифт SF Pro, regular 400, uppercase, radius 5px.
- Gap 6px/11px между клавишами/рядами (iOS-стандарт).
- Return **синий** `#0a84ff`, ⌫ **тёмно-серый** `#4a4a4c`.
- Статусы Wordle (correct/present/absent) переведены на iOS-палитру: `#2a7a5e` / `#a37c1a` / `#2c2c2e`.
- Popup альтернатив тоже в iOS-стиле.

### 3. Long-press на «Ь» → выбор «Ь/Ъ»
Добавил вторую клавишу с альтернативами. Сначала добавил «ъ» отдельной кнопкой (вариант 1), потом по фидбеку убрал — «ъ» выбирается удержанием «ь» через тот же механизм что и «е→ё». Индикатор-точка у «ь» появляется автоматически через `.wrd-key--has-alt::after`.

### 4. 🚨 КРИТФИКС: словарь загадок брался не из curated файла
**Пользователь заметил**: в игре попадались книжные/архаичные слова, хотя мы 2026-04-20 чистили `game-words.json`. Нашёл причину:

- Комментарий в коде говорил «только curated-список», а на деле грузился гигантский словарь с книжной лексикой.
- Curated `api/game-words.json` **вообще не использовался** для загадок (хотя в CLAUDE.md было написано обратное).

**Фикс**:
- Вернул seed из `require('../game-words.json')` в обоих драйверах.
- Добавил сброс `game_current_word / game_word_status / game_session_id` у активных игроков, если их текущее слово больше не входит в curated-список (таких после sync заставит получить новое слово при следующем `/api/game-stats`).
- `loadFiveLetterWords()` оставил как есть — он нужен для **валидации введённых гессов** (VALID_WORDS Set, 6074 слова).

## Решения
- **Drag-to-select vs tap-выбор**: iOS делает drag-to-select, но это агрессивно конфликтует с браузерным скроллом через touch. На нашей клавиатуре с маленькими кнопками лучше tap-выбор — надёжнее на всех устройствах.
- **Не использовать `loadFiveLetterWords()` для загадок**: curated — это то, чем управляет команда; 6074 пакетных слов — для валидации (там много устаревшего, но игрок-вводящий имеет право ошибиться, и это нормально).
- **Сброс активных игр при sync**: когда словарь меняется, у юзеров может быть `game_current_word` которого больше нет в `game_word_dictionary`. Безопаснее сбросить и переназначить, чем оставлять «осиротевшие» игры.
