## Правило выпуска новой версии

1. Обновить `src/version.js` → `APP_VERSION`
2. Обновить `package.json` → `version`
3. Добавить запись в `CHANGELOG.md`
4. `git tag vX.Y.Z && git push origin main --tags`

## Семантика версий

- **Patch** (1.0.X) — исправления багов
- **Minor** (1.X.0) — новые фичи без breaking changes
- **Major** (X.0.0) — крупные архитектурные изменения
