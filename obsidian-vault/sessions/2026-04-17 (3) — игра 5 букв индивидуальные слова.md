# 2026-04-17 (3) — Игра «5 букв»: индивидуальное слово для каждого пользователя

## Что сделано

### Архитектурное изменение: одно слово на день → личное слово

**Было**: таблица `game_daily_word` — одно слово для всех пользователей в день.

**Стало**: у каждого пользователя своё слово в `users.game_current_word`. Фиксируется до конца игры (победа или проигрыш), потом назначается новое при следующем заходе.

### Новые колонки в `users`

- `game_current_word TEXT` — загаданное слово для этого юзера
- `game_word_status TEXT` — `'active'` / `'won'` / `'lost'` / NULL
- `game_session_id TEXT` — уникальный ID сессии, используется как ключ localStorage на фронте

Runtime-миграции добавлены в `db.js` (try/catch) и `db-pg.js` (IF NOT EXISTS).

### Новые функции в `db.js` / `db-pg.js`

- `assignUserWord(userId)` — выбирает случайное слово из словаря, записывает в user, возвращает `{ word, sessionId }`
- `setUserWordStatus(userId, status)` — меняет `game_word_status`

### `api/game-stats.js`

- Если `game_word_status !== 'active'` → вызывает `assignUserWord` → назначает новое слово
- Возвращает `sessionId` в ответе

### `api/game-guess.js`

- Берёт слово из `user.game_current_word` вместо `getGameDailyWord()`
- На победу: `setUserWordStatus('won')` + `recordGameWin`
- На проигрыш (6-я попытка): `setUserWordStatus('lost')` + `reveal`

### Фронтенд

- `useWordle.js`: параметр `gameDay` → `sessionId`, ключ localStorage `wordle_progress_{sessionId}`
- `WordlePage.js`: передаёт `gameStats.sessionId` в `useWordle`
