# 2026-04-13 — Доставка с дозаказом, имя при регистрации

## Что сделано да, давай покажи. 

### 1. CheckoutForm — доставка при дозаказе с подарком
Раньше: любой подарок в корзине → доставка заблокирована, только самовывоз.
Теперь: если подарок + дозаказ → доступны и доставка, и самовывоз. Только подарок без дозаказа → самовывоз принудительно.

Логика: добавлены `hasNonGiftItems` и `hasOnlyGiftItems`. Блокировка и принудительный сброс в pickup работают только при `hasOnlyGiftItems`.

При доставке с подарком показывается подсказка «Подарок по подписке будет включён в ваш заказ».

### 2. CheckoutForm — fix автозаполнения адреса
Баг: если последний заказ был самовывозом, в localStorage сохранялся только `{ pickupPoint }`. При следующем открытии `saved` был truthy → `profile.last_address` из БД не читался → адрес не подставлялся.

Фикс: `(localData?.street ? localData : null) || profileAddr` — localStorage берётся только если содержит `street`, иначе читается `profile.last_address`.

### 3. LoginPage + set-password — запрос имени при регистрации
Новые пользователи (телефон → OTP → set-password) создавались с `name: null`.

Теперь: на шаге `set-password` для новых пользователей (`isNewUser === true`) показывается поле «Ваше имя» (обязательное). При сбросе пароля через «Забыли пароль?» (`isNewUser === false`) поле не показывается.

Бэкенд `set-password.js`: принимает `name`, передаёт в `upsertUser` при создании нового пользователя и при обновлении существующего.
