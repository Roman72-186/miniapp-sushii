# 2026-04-10 (3) — Форма добавления нового товара в AdminPage

## Что сделано

- **Новый таб `⊞ Товар+`** в группе «Контент» AdminPage
- Форма позволяет добавить товар без git, rebuild и ручного редактирования JSON
- **Фото** загружается через браузер → сохраняется в `data/product-images/{slug}.{ext}` (persistent volume)
- **Поля**: категория (роллы/запечённые/сеты), название, артикул FP, цена без скидки, описание, фото
- **Опциональные каталоги**: чекбокс «В скидочный магазин» (+ отдельный SKU) и «В подарочные» (только для роллов и сетов, + SKU)
- В подарочном каталоге цена автоматически устанавливается в 0
- `DiscountShopPage`: приоритет явного поля `item.image` из JSON над `imageMap.js` — новые товары с загруженным фото отображаются без изменения imageMap

## Архитектурные решения

- Фото хранится в `data/product-images/` (persistent volume на VPS), не в `public/` и не в git
- Slug формируется как `name.toLowerCase().replace(/ё/г,'е')...replace(/\s+/g,'-')` — без спецсимволов
- Кэш изображений: `public, max-age=604800` (как у остальных картинок сайта)
- Экспорт утилит из `admin-products.js`: `handler` отдельно + `readCatalog`/`saveCatalog`/`CATALOGS` через `module.exports.X`
