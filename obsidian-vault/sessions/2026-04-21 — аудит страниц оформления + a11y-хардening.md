# 2026-04-21 — аудит страниц оформления + a11y-хардening

## Что сделано
Технический аудит (`/audit`) страниц оформления заказа + прогон всех рекомендаций (harden + adapt + clarify + quieter + optimize + polish одним блоком).

Начальный скор: **11/20 (Acceptable)**. Основные дыры — a11y (`htmlFor`, aria), тач-таргеты <44px, отсутствие токенов, бесконечная анимация `gift-glow`.

## Изменённые файлы
- `src/components/CheckoutForm.js` — id/htmlFor/required/aria-required на все инпуты, autoComplete, role="combobox" на street-autocomplete, role="listbox"/option на дропдауне, error в role="alert"+aria-live, aria-busy на submit, убраны инлайн-стили
- `src/components/SubCheckoutModal.js` — аналогично (id/htmlFor/required), role="dialog"+aria-modal на модалке, убраны inline style (`zIndex`, preview-стили)
- `src/components/CartPanel.js` — `useMemo(cartSkus)`, role="dialog"+aria-modal, aria-label на close/remove/± с именем товара, label для промокода, aria-invalid/aria-describedby/aria-live
- `src/shop.css` — блок `:root` с токенами (`--shop-accent`, `--shop-error`, `--shop-tap-min: 44px`, `--shop-focus-ring`), глобальный `*:focus-visible` box-shadow, `@media (prefers-reduced-motion: reduce)`, подняты тач-таргеты (`shop-cart__close`, `shop-cart__item-btn` 28→44, `shop-cart__item-remove`, `shop-checkout__back`), убран градиент `#2b1500→#1a2018` у gift-section (плоский `#1f1a14`), `gift-glow` больше не `infinite` — три пульса (`ease-in-out 3`), новые классы `shop-form-field__required`, `--confirm`, `--relative`, `shop-form-row--spaced`, `shop-form-error`, `shop-checkout--modal`, `shop-gift-preview`, `shop-cart__promo-label`
- `src/shop-v2.css` — `shop-cart__item-remove` 30→44px, `shop-cart__promo-clear` 32→44px + `:focus-visible`

## Ключевые решения
- **Токены, а не глобальный ребрендинг**: ввёл CSS custom properties, но заменил хардкоды только в новых стилях. Старые места живут, но будущие правки пойдут через токены. Постепенная миграция без риска регрессий.
- **`:focus-visible` через box-shadow, а не outline** — в магазине многое имеет `border-radius`, outline режется. Тёмно-зелёный ring `rgba(60,200,161,0.45)` контрастит на тёмном фоне.
- **`gift-glow` не полностью убрал, а ограничил тремя пульсами** — визуальный призыв на появлении подарка сохранён, но без бесконечного раздражителя.
- **Error-блок — постоянный контейнер с `:empty { display: none }`** вместо условного рендера. Так `aria-live` регион всегда в DOM, скринридер нормально объявляет появление ошибки.
- **`role="combobox"` на street-autocomplete** — ESLint ругался на `aria-expanded` у implicit textbox; переход на combobox-паттерн с `aria-controls` на listbox — корректно по WAI-ARIA.
