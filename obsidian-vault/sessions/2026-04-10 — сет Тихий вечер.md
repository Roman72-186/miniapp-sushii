## Ловушки, с которыми столкнулись

### Docker кэширует COPY слои
При `docker compose up -d --build` — если код не запушен на git, `git pull` ничего не подтянет, и Docker использует кэшированный слой с `COPY public/ ...`. В результате `build/` содержит старые файлы. **Решение: всегда git push перед деплоем.**

### localhost:3001 ≠ контейнер
После пересборки контейнер не публикует порт 3001 наружу (только внутри Docker network). `curl localhost:3001` шёл к старому контейнеру/iptables и возвращал устаревшие данные. Реальная проверка — через `curl https://sushi-house-39.ru/...` или через IP контейнера.
