# 2026-04-15 — Допродажи в корзине

## Что сделано

Реализована полная система допродаж (upsell) в корзине с управлением из AdminPage.

### Новые файлы
- `api/upsell-items.js` — публичный GET, ищет товары по SKU из upsell.json во всех 10 каталогах
- `api/admin/upsell-toggle.js` — POST (admin auth), add/remove SKU, макс 6 товаров
- `api/admin/upsell-clear.js` — POST (admin auth), очистить весь список
- `src/components/UpsellBlock.js` — блок «Добавьте к заказу» в корзине, до 4 товаров

### Изменённые файлы
- `server.js` — регистрация 3 новых роутов
- `src/components/CartPanel.js` — prop `onAddItem`, рендер `<UpsellBlock />`
- `src/ShopPage.js` — передаёт `onAddItem={cart.addItem}`
- `src/DiscountShopPage.js` — передаёт `onAddItem={cart.addItem}` (был баг — не передавалось)
- `src/AdminPage.js` — таб «◈ Допродажи», кнопки ☆/★ в табе «Товары»
- `src/shop.css` — стили для `.upsell-block`

### Хранилище
`data/products/upsell.json` — массив SKU, персистентный в Docker volume

## Ключевые решения

- **Хранилище**: отдельный файл `data/products/upsell.json` (паттерн override-файлов)
- **Поиск SKU**: во всех 10 каталогах (изначально только 3 — был баг, SKU не находились)
- **Фильтрация**: UpsellBlock исключает товары уже в корзине, deps `[JSON.stringify(cartSkus)]`

## Баги по пути

1. **Авторизация upsell-toggle была отключена** — `checkAuth` был закомментирован implementer'ом, восстановлен вручную
2. **upsell-items не регистрировался через `app.get`** — при curl возвращал HTML. Оказалось routes работают, но curl обращался через nginx который кэшировал старый ответ
3. **upsell.json заполнился SKU из rolls-sub** (1105-1146) — эти SKU не находились в 3 каталогах. Очищен вручную + расширен поиск на все каталоги
4. **DiscountShopPage не передавала onAddItem** — UpsellBlock никогда не показывался. Однострочный фикс
5. **cartSkus не в deps useEffect** — UpsellBlock не обновлялся при изменении корзины. Исправлен
