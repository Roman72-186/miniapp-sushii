## Ловушки

### Docker кэширует COPY слои даже после git pull
`docker compose up -d --build` использует кэшированный слой `COPY . .` если образ уже существует. Новые файлы не попадают в контейнер. **Решение: всегда `docker compose build --no-cache app` при изменении кода.**
