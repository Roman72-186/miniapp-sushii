## Docker Compose

Три сервиса:

```
app      ← Node 20-alpine, порт 3001
nginx    ← reverse proxy + SSL termination
certbot  ← Let's Encrypt, авторенью
```

**Persistent volume**: `app-data` → `/app/data`
Содержит: `sushii.db`, `users/`, `gifts/`, `products/`, `banners/`

> ⚠️ Все данные пользователей, платежей и кэши — в этом volume. При пересборке контейнера данные сохраняются.

## Деплой (команды)

```bash
# Подключиться
ssh root@64.188.63.249

# Задеплоить
cd miniapp-sushii && git pull && docker compose up -d --build

# Посмотреть логи
docker compose logs -f app

# Рестартовать без пересборки
docker compose restart app
```

## Nginx

Reverse proxy на порт 3001. Раздаёт статику React (`build/`) и проксирует `/api/*` на Node.

SSL-сертификаты через Certbot (Let's Encrypt). Автоматическое обновление.

## CORS

Переменная `ALLOWED_ORIGIN` (по умолчанию `https://sushi-house-39.ru`).
Каждый API-обработчик выставляет заголовки вручную (нет общего CORS middleware).

## Cron подписок

Запускается внутри `server.js` через `setTimeout`. Каждый день в 10:00 UTC (13:00 МСК):
- Проверяет истёкшие подписки
- Пробует рекуррентное списание через YooKassa
- Деактивирует если не удалось

Также доступен через `POST /api/cron-subscriptions` с заголовком `Authorization: Bearer {CRON_SECRET}` — для ручного запуска или внешнего cron.
