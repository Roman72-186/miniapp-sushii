# Архитектура приложения

## Общая схема

```
Telegram Bot  ──→  кнопка "Открыть"
                        ↓
              Telegram WebApp (Mini App)
                        ↓
              React SPA (CRA, port 3000 dev)
                        ↓
              Express API (port 3001)
                        ↓
              SQLite (data/sushii.db)
```

## Frontend (`src/`)

**Роутинг** — без react-router. В `App.js` читается `window.location.pathname` и рендерится нужный компонент:

| Путь | Компонент | Назначение |
|------|-----------|-----------|
| `/` | `LandingPage` | Тарифы, кнопка подписки |
| `/discount-shop` | `DiscountShopPage` | Скидки + подарки (главная для подписчиков) |
| `/shop` | `ShopPage` | Обычный магазин |
| `/profile` | `ProfilePage` | Личный кабинет |
| `/settings` | `SettingsPage` | Настройки |
| `/pay/:id` | `PaymentPage` | Оплата подписки |
| `/login` | `LoginPage` | Веб-вход по телефону |
| `/benefits` | `BenefitsPage` | Выгода подписки |
| `/partner-code` | `PartnerCodePage` | Ввод кода партнёра |
| `/admin` | `AdminPage` | Админка |
| `/gift-rolls` | `GiftRollsPage` | Подарочные роллы (бот) |
| `/gift-sets` | `GiftSetsPage` | Подарочные сеты (бот) |
| `/sets-received` | `SetsReceivedPage` | Сет уже получен |
| `/sets`, `/rolls` | `SetsPage`, `RollsPage` | Legacy |

**Глобальный стейт** — `UserContext.js` (`useUser()` хук). Поля: `telegramId`, `tarif`, `phone`, `profile`, `sync()`, `hasTag()`, `isWebUser`.

**Стили** — plain CSS, `shop.css` и `App.css`. Тёмная тема: `#1a1a1a` (фон), `#2a2a2e` (карточки), акцент `#3CC8A1`.

**Корзина** — `src/hooks/useFrontpad.js` → `useCart()`. Подарки нельзя дублировать и изменять количество.

## Backend (`server.js` + `api/`)

Express на порту 3001. Каждый API-эндпоинт — отдельный файл в `api/`. Утилиты в `api/_lib/`.

**Порядок отдачи статики (первое совпадение побеждает):**
1. `data/banners/` — баннеры, загруженные через админку
2. `data/products/` — переопределения цен/флагов от админки
3. `public/admin/` → путь `/admin` (до React build!)
4. `build/` — собранный React

→ подробнее: [[приоритет статических файлов в Express]]

## Поток данных пользователя

1. Фронт вызывает `POST /api/sync-user` с `{ telegram_id, tg_name, force }`
2. Бэк читает SQLite → `deriveFromDbUser()` → пишет в `data/users/{id}.json`
3. Кэш живёт 5 минут, при `force=true` перезаписывается сразу
4. `UserContext` получает данные и раздаёт компонентам
