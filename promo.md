Нужно доработать существующее React веб-приложение доставки суши, роллов и сетов.

Контекст:
- проект уже существует;
- это React веб-приложение;
- корзина уже реализована;
- блок допродаж уже реализован, его трогать не нужно;
- текущую механику допродаж менять не нужно;
- не нужно переписывать корзину целиком;
- использовать существующие состояния, хуки, методы addToCart / updateCart / applyPromo / recalculation, если они уже есть.

Нужно реализовать 2 механики подарочного ролла:
1. Подарок по промокоду при достижении минимальной суммы заказа
2. Автоматический подарок при заказе от 2500 ₽ и выше

==================================================
1. Подарок по промокоду
==================================================

Бизнес-логика:
- пользователь вводит промокод в корзине;
- промокод для подарка: 102030;
- если сумма товаров в корзине >= 2000 ₽, автоматически добавляется подарочный ролл;
- подарок бесплатный;
- количество подарка = 1;
- подарок должен быть визуально помечен в корзине:
  - "Подарок"
  - "Бесплатно"
  - цена 0 ₽
- если сумма становится меньше 2000 ₽, подарок удаляется автоматически;
- если промокод удалён, сброшен или стал невалидным, подарок удаляется автоматически;
- подарок не должен участвовать в расчёте порога для самого себя;
- подарок нельзя дублировать повторным вводом промокода.

Важно:
- роллы, которые можно выдавать по промокоду, должны настраиваться в админке у товара;
- для этого у товара нужно добавить отдельный флаг / кнопку:
  - "Подарок по промокоду"
  или аналогичное системное поле.

Логика выбора подарка по промокоду:
- если в админке отмечен один товар как подарок по промокоду — выдавать его;
- если отмечено несколько товаров — выбрать один из них по понятному правилу;
- по умолчанию можно реализовать:
  - либо первый активный товар,
  - либо случайный активный товар;
- если выбирается случайно, выбор должен фиксироваться для текущей корзины и не меняться при каждом пересчёте.

==================================================
2. Автоматический подарок при заказе от 2500 ₽
==================================================

Бизнес-логика:
- если сумма товаров в корзине >= 2500 ₽, автоматически добавляется подарочный ролл;
- подарок бесплатный;
- количество подарка = 1;
- подарок должен быть визуально помечен в корзине;
- если сумма становится меньше 2500 ₽, подарок удаляется автоматически;
- подарок не должен участвовать в расчёте порога для самого себя;
- подарок не должен дублироваться.

Важно:
- роллы, которые могут выдаваться как подарок при заказе от 2500 ₽, должны выбираться в админке;
- для этого у товара нужно добавить отдельный флаг / кнопку:
  - "Подарок от 2500"
  или аналогичное системное поле.

Логика выбора:
- если отмечен один товар — выдавать его;
- если отмечено несколько товаров — выбирать подарок случайным образом;
- если товаров больше двух, всё равно выбирать случайным образом из всех доступных;
- случайный выбор должен происходить один раз для текущей корзины / текущего заказа;
- после выбора подарок не должен меняться при каждом изменении количества товаров;
- новый случайный выбор допускается только если:
  - подарок был удалён из-за потери порога,
  - корзина была очищена,
  - создан новый заказ / новая сессия корзины.

==================================================
4. Совместимость двух механик
==================================================

Нужно поддержать обе механики одновременно:
- подарок по промокоду при сумме от 2000 ₽;
- подарок при сумме от 2500 ₽.

Поведение:
- если заказ подходит только под одну механику — добавить один соответствующий подарок;
- если заказ подходит под обе механики одновременно — поддержать оба подарка, если это допускается бизнес-логикой;
- каждый источник подарка должен добавлять не более одного подарка;
- нельзя допускать бесконечного дублирования.

Безопасное правило реализации:
- максимум один подарок на каждый тип механики:
  - 1 подарок по промокоду;
  - 1 подарок по порогу 2500;
- если по обеим механикам выбран один и тот же товар, не создавать два одинаковых line item автоматически без необходимости;
- в таком случае:
  - либо объединить в один gift line,
  - либо оставить один подарок,
  - выбрать наиболее безопасный вариант для текущей архитектуры и явно описать его.

- промокод удалён;
- промокод невалиден;
