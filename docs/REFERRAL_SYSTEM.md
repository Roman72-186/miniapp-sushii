# Реферальная система Суши-Хаус 39

## Общее описание

Реферальная система состоит из двух частей:
1. **Для всех пользователей** — приглашай друзей, получай бонусы SHC
2. **Для амбассадоров** — двухуровневая комиссия с платежей рефералов (30% / 5%)

Вся логика работает **в мини-аппе** (SQLite), независимо от WATBOT.

---

## Часть 1: Рефералка для обычных пользователей

### Как работает
1. У каждого пользователя есть **реферальная ссылка**
2. Пользователь делится ссылкой через кнопку "Поделиться ссылкой" в Личном кабинете
3. Друг переходит по ссылке → регистрируется в боте
4. Пользователю **автоматически начисляется 50 SHC**
5. На определённых порогах — дополнительный бонус

### Начисление SHC

**Базовое:** +50 SHC за каждого приглашённого друга

**Бонусы на порогах:**

| Друзей | Бонус | Итого получит за порог |
|--------|-------|-----------------------|
| 1 | +50 SHC | 100 SHC |
| 3 | +100 SHC | 150 SHC |
| 5 | +290 SHC | 340 SHC |
| 10 | +490 SHC | 540 SHC |
| 50 | +1 500 SHC | 1 550 SHC |
| 100 | +5 000 SHC | 5 050 SHC |
| 500 | +10 000 SHC | 10 050 SHC |
| 1000 | +30 000 SHC | 30 050 SHC |

### Что видит пользователь в Личном кабинете
- Счётчик **друзей** и **баланс SHC** (два блока)
- Кнопка **"Поделиться ссылкой"**
- **История начислений SHC** — за кого начислено, сколько базовых + бонус, дата
- Подсказка: "+50 SHC за каждого друга. Бонусы на порогах: 5, 10, 50, 100 друзей"
- Блок **"Стать амбассадором — 9 990 ₽"** (для перехода на амбассадорскую программу)

### Статус: полностью реализовано ✅
- Начисление 50 SHC за нового реферала ✅
- Бонусы на порогах (8 уровней) ✅
- Защита от повторного учёта ✅
- Достижения (текстовые) ✅
- История начислений SHC в кабинете ✅

## Часть 2: Амбассадорская программа

### Как работает
1. Пользователь покупает тариф **9 990₽** (одноразовый платёж)
2. Получает статус **Амбассадор** и реферальную ссылку
3. Делится ссылкой — приглашает людей
4. Когда приглашённый покупает подписку (290 / 490 / 1190₽) — амбассадору **автоматически начисляется 30%**

### Уровни дохода

| Уровень | Комиссия | Условие |
|---------|----------|---------|
| **Level 1** | 30% от платежей рефералов | Сразу после покупки тарифа 9990₽ |
| **Level 2** | 5% от платежей рефералов своих амбассадоров | Разблокируется при 10+ амбассадорах в команде |

### Примеры начислений

**Level 1 (30%):**
| Реферал оплатил | Амбассадор получает |
|-----------------|---------------------|
| 290₽ | 87₽ |
| 490₽ | 147₽ |
| 1 190₽ | 357₽ |

**Level 2 (5%) — пример:**
Амбассадор "А" пригласил 10 амбассадоров. Один из них ("Б") пригласил реферала, который оплатил 490₽:
- Амбассадор "Б" получает 30% = **147₽**
- Амбассадор "А" получает 5% = **24.50₽**

### Что видит амбассадор в Личном кабинете
- Золотой бейдж **"АМБАССАДОР"**
- Счётчики: **рефералов** / **амбассадоров** среди них
- **Прогресс-бар Level 2** (X / 10 амбассадоров для разблокировки)
- Блок **"Заработок"** — всего ₽ / Level 1 / Level 2
- **История начислений** — имя реферала, сумма платежа, комиссия, процент, дата
- **Баланс к выплате** (в рублях)
- Кнопка **"Поделиться ссылкой"**

### Статус: полностью реализовано ✅
- Автоматическое начисление 30% при оплате реферала ✅
- Level 2 (5%) при ≥10 амбассадорах ✅
- Запись всех платежей и комиссий в БД ✅
- История и баланс в кабинете ✅

## Что осталось сделать

### 1. Привязка реферальной ссылки к мини-аппу (критично)
**Сейчас:** когда пользователь переходит по реферальной ссылке, WATBOT регистрирует реферала у себя. Но для начисления SHC и комиссий нужно, чтобы связь записалась в нашу SQLite базу.

**Нужно:** при переходе по ссылке вызывать API мини-аппа:
```
POST /api/register-referral
{ "telegram_id": "id_нового_пользователя", "invited_by": "id_пригласившего" }
```

**Варианты:**
- **В боте:** при старте по реферальной ссылке бот делает HTTP-запрос на `/api/register-referral`
- **В мини-аппе:** парсить параметр `ref` из URL при первом входе

### 2. Магазин за SHC (нужно решение)
Пользователи копят SHC, но пока не могут их потратить.

**Варианты (нужно выбрать):**
- Скидка на заказ (1 SHC = 1₽?)
- Бесплатное блюдо за N SHC
- Обмен на рубли
- Оплата подписки

### 3. Механизм вывода средств для амбассадоров
**Сейчас:** "Для вывода средств свяжитесь с администратором" (ручной).

**В будущем:**
- Кнопка "Запросить выплату"
- Минимальная сумма для вывода
- Автоматический перевод (опционально)

### 4. Уведомления
**Не реализовано.** При начислении SHC или комиссии пользователь не получает push-уведомление.

**Можно добавить:**
- Сообщение в Telegram-бот: "Вам начислено 147₽ за оплату реферала Иван"
- Или: "+50 SHC за нового друга @username"

### 5. Полный отказ от WATBOT
**Текущее состояние:**
- Рефералы, SHC, комиссии — **полностью в SQLite** ✅
- WATBOT по-прежнему используется для:
  - Синхронизации контактных данных (имя, телефон)
  - Реферальных ссылок (ref_url генерируется в WATBOT)
  - Тегов (определение тарифа при sync-user)
  - Рассылок через бота

**Для полного отказа:**
- Генерировать реферальные ссылки самостоятельно
- Хранить все данные пользователя только в SQLite
- Оставить WATBOT только для рассылок

## Техническая архитектура

### Стек
- **БД:** SQLite (better-sqlite3), файл `/app/data/sushii.db`
- **Бэкенд:** Express.js на Node 20
- **Фронт:** React (CRA)
- **Деплой:** Docker + Nginx + Let's Encrypt на VPS (64.188.63.249)
- **Оплата:** YooKassa (webhook обрабатывает начисления)

### API эндпоинты реферальной системы
| Эндпоинт | Назначение |
|----------|-----------|
| `POST /api/register-referral` | Регистрация связи + начисление SHC |
| `POST /api/get-referrals` | Список рефералов пользователя |
| `POST /api/get-transactions` | История: комиссии (₽) + SHC бонусы |
| `POST /api/yookassa-webhook` | Обработка платежей + начисление комиссий |
| `POST /api/migrate-referrals` | Одноразовая миграция из WATBOT |

### Схема базы данных

```
users
├── telegram_id (PK)
├── name, phone, tariff
├── invited_by → telegram_id пригласившего
├── balance_shc (общий баланс: SHC + комиссии)
├── is_ambassador (0/1)
├── subscription_status, subscription_start, subscription_end
└── created_at, updated_at

payments (все платежи через YooKassa)
├── id (PK)
├── telegram_id → users
├── tariff, amount, months
├── yookassa_payment_id
└── created_at

transactions (комиссии амбассадоров, ₽)
├── id (PK)
├── ambassador_id → users
├── referral_id → users
├── payment_amount, commission_amount, commission_percent
├── level (1 или 2)
└── created_at

referral_bonuses (SHC бонусы за друзей)
├── id (PK)
├── user_id → users (кому начислено)
├── referral_id → users (за кого)
├── base_amount (50), threshold_bonus, total_amount
├── friends_count, achievement
└── created_at
```

### Ключевые файлы
| Файл | Назначение |
|------|-----------|
| `api/_lib/db.js` | SQLite — таблицы, CRUD, логика комиссий и SHC |

| `api/register-referral.js` | Регистрация реферала + начисление SHC |
| `api/get-referrals.js` | Список рефералов из SQLite |
| `api/get-transactions.js` | История начислений (комиссии + SHC) |
| `api/sync-user.js` | Синхронизация WATBOT → SQLite |
| `src/ProfilePage.js` | Личный кабинет (заработок, SHC, рефералы) |
