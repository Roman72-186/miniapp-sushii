# Подарочные страницы

## 1. Страница GiftRollsPage (`/gift-rolls`)

**Назначение:** выбор подарочного ролла для подписчиков тарифа 490₽.

**Как открывается:** через кнопку в Telegram-боте. Бот формирует ссылку на мини-апп с маршрутом `/gift-rolls`.

**Поток:**

1. Пользователь открывает страницу через кнопку в боте
2. Загружается список доступных роллов из `/подписка 490/rolls-490.json`
3. Пользователь выбирает один ролл
4. Заполняет форму: имя, телефон, пункт самовывоза (автозаполнение из профиля)
5. Нажимает "Подтвердить" — заказ отправляется в Frontpad через `/api/create-order`
6. Параллельно вызывается `/api/claim-gift` для фиксации получения подарка
7. Бот отправляет подтверждение в чат через `/api/send-bot-message`
8. Мини-апп автоматически закрывается (`Telegram.WebApp.close()`)

**Доступ:** только тариф 490₽.

**Периодичность:** раз в 15 дней (управляется системой подарочных окон).

**Файл:** `src/GiftRollsPage.js`

## 3. Раздел подарков в DiscountShopPage (`/discount-shop`)

Страница скидочного магазина содержит раздел подарков в верхней части — кнопки для перехода к выбору подарочных роллов и сетов.

### Кнопки подарков

Отображаются в блоке `shop-gift-row`. Каждая кнопка соответствует категории из `GIFT_CATEGORIES`:

- **Роллы в подарок** (`gift-rolls`) — доступно для тарифа 490₽
- **Сеты в подарок** (`gift-sets`) — доступно для тарифа 1190₽

### Визуальные состояния кнопок

| Состояние | Описание | Внешний вид |
|-----------|----------|-------------|
| Загрузка | Тариф или статус подарка ещё загружается | Кнопка заблокирована, бейдж `...` |
| Заблокирован | Тариф пользователя не соответствует | Кнопка заблокирована, бейдж 🔒 |
| Доступен | Подарок можно выбрать | Кнопка активна |
| Получен (`claimed`) | Подарок уже получен в текущем периоде | Текст "✓ Получен", стиль `--claimed` |
| Ожидание (`waiting`) | До следующего подарка осталось N дней | Текст "Через N дн.", стиль `--waiting` |
| В корзине | Подарок уже добавлен в корзину | Текст "✓ В корзине" или "Уже выбран" |

### Поток выбора подарка в магазине

1. Пользователь нажимает кнопку подарка
2. Проверяется доступ (тариф, статус, наличие подарка в корзине)
3. При доступности — открывается список товаров (view `gift-rolls` или `gift-sets`)
4. Пользователь выбирает один подарок — он добавляется в корзину
5. Возврат на основную страницу магазина
6. Пользователь может добавить платные товары со скидкой
7. При оформлении заказа подарок фиксируется через `/api/claim-gift`

**Файл:** `src/DiscountShopPage.js`

## 4. Система подарочных окон

### Назначение

Система управляет периодичностью получения подарков. Каждый подписчик может получить подарок один раз за период (окно). Периоды рассчитываются от даты начала подписки.

### Как работает `gift-windows.js`

**Файл:** `api/_lib/gift-windows.js`

Основные функции:

- **`buildWindows(startDate, endDate, periodDays)`** — генерирует массив временных окон между датой начала и конца подписки. Каждое окно имеет `from` и `to` (даты начала и конца периода).
- **`getCurrentWindow(windows)`** — находит текущее активное окно на основе сегодняшней даты.
- **`computeStatus(windows, claimedDates)`** — вычисляет статус подарка:
  - `available` — подарок доступен для получения
  - `claimed` — подарок уже получен в текущем окне
  - `waiting` — до начала следующего окна осталось N дней
  - `expired` — все окна исчерпаны

Вспомогательные функции: `parseDDMMYYYY`, `formatDDMMYYYY`, `addDays`, `todayUTC`.

### Структура хранилища (blob-store)

Данные о полученных подарках хранятся в blob-store. API `/api/get-gift-windows` возвращает:

```json
{
  "success": true,
  "data": {
    "currentStatus": "available|claimed|waiting|expired",
    "daysLeft": 5,
    "currentWindow": 2,
    "totalWindows": 12,
    "adminGrants": { "roll": false, "set": false }
  }
}
```

API `/api/claim-gift` фиксирует получение подарка, записывая дату в массив `claimedDates`.

### Расчёт периодов

- Тариф 490₽ (роллы): период 15 дней
- Тариф 1190₽ (сеты): период 30 дней
- Окна рассчитываются от `subscription_start` до `subscription_end`
- Админ-гранты (`adminGrants`) позволяют выдать подарок вне расписания
