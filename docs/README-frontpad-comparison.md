# Сравнение товаров с Frontpad

Этот скрипт (`compare-frontpad-products.js`) предназначен для сравнения товаров из локальных JSON-файлов с данными из Frontpad API по цене и идентификаторам (SKU).

## Возможности

- Сравнение всех товаров из локальных JSON-файлов с товарами из Frontpad
- Определение товаров, которых нет в Frontpad
- Определение товаров из Frontpad, которых нет в локальных JSON
- Выявление расхождений в ценах
- Выявление расхождений в названиях
- Генерация CSV-отчета для дальнейшего анализа

## Требования

1. Node.js (версия 18+ рекомендуется)
2. Настроенный файл `.env` с переменной окружения `FRONTPAD_SECRET`

## Использование

1. Убедитесь, что у вас настроен файл `.env` с API ключом Frontpad:

```
FRONTPAD_SECRET=ваш_секретный_ключ_frontpad
```

2. Запустите скрипт:

```bash
node compare-frontpad-products.js
```

3. После выполнения скрипт:
   - Выведет в консоль статистику и список расхождений
   - Создаст файл `frontpad-comparison.csv` с подробным отчетом

## Анализ результатов

### Консольный вывод

Скрипт выводит следующую информацию:

- Количество загруженных товаров из JSON и Frontpad
- Статистику совпадений и расхождений
- Список товаров из JSON, которых нет в Frontpad (с разделением на обычные и подписочные)
- Список расхождений в ценах (с разделением на обычные и подписочные товары)
- Список расхождений в названиях товаров
- Список товаров из Frontpad, которых нет в JSON (первые 20 записей)

### CSV-отчет

Файл `frontpad-comparison.csv` содержит следующие данные:

- Тип расхождения (отсутствие во Frontpad, отсутствие в JSON, расхождение в цене)
- SKU товара
- Название в JSON
- Цена в JSON
- Источник (путь к JSON-файлу)
- Название во Frontpad
- Цена во Frontpad
- Разница в цене

## Интерпретация результатов

1. **Товары из JSON, которых нет в Frontpad**
   - Эти товары могут потребовать добавления в Frontpad или корректировки SKU

2. **Расхождения в ценах**
   - Для обычных товаров: требуется синхронизация цен между системами
   - Для товаров подписки: расхождения могут быть нормальными, так как скидки применяются на фронтенде

3. **Расхождения в названиях**
   - Требуется стандартизация названий между системами
   - Обратите внимание на форматирование (пробелы, заглавные буквы, специальные символы)

4. **Товары из Frontpad, которых нет в JSON**
   - Эти товары могут требовать добавления в JSON-файлы или указывать на устаревшие SKU

## Настройка

При необходимости вы можете настроить скрипт, изменив список JSON-файлов для проверки в переменной `jsonFiles` в начале скрипта.

## Важные замечания

- Скрипт сравнивает SKU (идентификаторы товаров) между системами
- Для корректной работы SKU должны быть согласованы между локальными JSON и Frontpad
- Скрипт чувствителен к форматированию - в названиях учитываются заглавные/строчные буквы и пробелы
