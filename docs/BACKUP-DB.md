# Резервное копирование базы данных

## 📋 Описание

Скрипт создаёт резервную копию SQLite базы данных в директорию `data/backups/`.

**Важно:** Копии **не удаляются автоматически** и хранятся до ручного удаления.

## 🚀 Использование

### Локально

```bash
npm run backup:db
```

### На сервере

```bash
cd /root/miniapp-sushii
node scripts/backup-db.js
```

### Через SSH

```bash
ssh root@sushi-house-39.ru "cd /root/miniapp-sushii && node scripts/backup-db.js"
```

## 📁 Структура хранения

```
data/
└── backups/
    ├── sushii.backup.2026-03-26_15-30-00.db
    ├── sushii.backup.2026-03-26_16-45-00.db
    └── sushii.backup.2026-03-27_10-00-00.db
```

Имя файла содержит дату и время создания в формате `YYYY-MM-DD_HH-MM-SS`.

## 🧹 Управление копиями

### Просмотр всех копий

```bash
ls -lh data/backups/
```

### Удаление старой копии

```bash
rm data/backups/sushii.backup.2026-03-25_10-00-00.db
```

### Удаление всех копий (осторожно!)

```bash
rm data/backups/*.db
```

### Автоматическая очистка (через cron)

Для автоматического удаления копий старше 7 дней добавьте в crontab:

```bash
# Удаление бэкапов старше 7 дней (каждый день в 3:00)
0 3 * * * find /root/miniapp-sushii/data/backups -name "sushii.backup.*.db" -mtime +7 -delete
```

## 🔄 Автоматизация

### Ежедневное резервное копирование

Добавьте в crontab (`crontab -e`):

```bash
# Ежедневный бэкап в 2:00 ночи
0 2 * * * cd /root/miniapp-sushii && node scripts/backup-db.js >> logs/backup.log 2>&1
```

### Перед обновлением приложения

```bash
# Деплой с бэкапом
cd /root/miniapp-sushii && node scripts/backup-db.js && git pull && docker-compose restart
```

## 📝 Рекомендации

1. **Создавайте копию перед:**
   - Обновлением приложения
   - Миграцией данных
   - Изменением схемы БД
   - Массовыми изменениями пользователей

2. **Храните копии:**
   - Минимум 7 дней
   - Рекомендуется 30 дней

3. **Проверяйте целостность:**
   ```bash
   sqlite3 data/backups/sushii.backup.2026-03-26_15-30-00.db "PRAGMA integrity_check;"
   ```

## 🔒 Безопасность

- Копии хранятся локально на сервере
- Рекомендуется настроить синхронизацию с облаком (S3, Google Drive)
- Ограничьте доступ к директории `data/backups/`

```bash
chmod 700 data/backups/
```
